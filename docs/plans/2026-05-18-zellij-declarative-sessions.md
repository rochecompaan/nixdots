# Zellij Declarative Sessions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a manual `zellij-start-sessions` command that creates declared Zellij sessions in order from native KDL layouts and then attaches to the first session.

**Architecture:** Keep each session layout as hand-written Zellij KDL. Add a small Nix manifest that names each session and points to its KDL file, then generate Home Manager config entries and a startup script from that manifest.

**Tech Stack:** Nix/Home Manager, Zellij 0.44.1 KDL layouts, POSIX shell generated by `pkgs.writeShellApplication`.

---

## File Structure

- `modules/home/desktop/shell/zellij/sessions.nix`
  - New ordered manifest of declared sessions.
  - Each item has `name` and `layout`.
  - List order defines startup order; first item is the attach target.
- `modules/home/desktop/shell/zellij/sessions/nixdots.kdl`
  - First concrete native Zellij session layout.
  - Proves the pattern with a known repo session and declared pane commands.
- `modules/home/desktop/shell/zellij/default.nix`
  - Imports the session manifest.
  - Installs session KDL files under `~/.config/zellij/layouts/sessions/`.
  - Generates and packages `zellij-start-sessions`.
- `docs/plans/2026-05-18-zellij-declarative-sessions.md`
  - This implementation plan.

## Important Zellij command detail

When the script is executed from inside an existing Zellij session, plain `--layout` adds the layout to the current session. Use `--new-session-with-layout` to force creation of a separate session:

```sh
zellij --session "$name" --new-session-with-layout "$layout" attach --create-background "$name"
```

This was smoke-tested during planning with a temporary session and Zellij created the requested background session.

### Task 3: Generate the manual startup script

**Files:**
- Modify: `modules/home/desktop/shell/zellij/default.nix`

- [ ] **Step 1: Add script-generation bindings to the `let` block**

In `modules/home/desktop/shell/zellij/default.nix`, after the `declarativeSessionLayouts` binding from Task 2, add:

```nix
  firstDeclarativeSession = (builtins.head declarativeSessions).name;
  createDeclarativeSessionCommands = pkgs.lib.concatMapStringsSep "\n" (session: ''
    create_session ${pkgs.lib.escapeShellArg session.name} ${pkgs.lib.escapeShellArg "${config.xdg.configHome}/zellij/layouts/sessions/${session.name}.kdl"}
  '') declarativeSessions;
  zellijStartSessions = pkgs.writeShellApplication {
    name = "zellij-start-sessions";
    runtimeInputs = [
      pkgs.gnugrep
      pkgs.gnused
      pkgs.zellij
    ];
    text = ''
      session_exists() {
        zellij list-sessions -n 2>/dev/null | sed 's/ \[Created.*//' | grep -Fxq "$1"
      }

      create_session() {
        name="$1"
        layout="$2"

        if session_exists "$name"; then
          echo "zellij-start-sessions: skipping existing session '$name'"
          return 0
        fi

        echo "zellij-start-sessions: creating session '$name' from $layout"
        zellij --session "$name" --new-session-with-layout "$layout" attach --create-background "$name"
      }

      ${createDeclarativeSessionCommands}

      if [ -n "''${ZELLIJ:-}" ]; then
        zellij action switch-session ${pkgs.lib.escapeShellArg firstDeclarativeSession}
      else
        exec zellij attach ${pkgs.lib.escapeShellArg firstDeclarativeSession}
      fi
    '';
  };
```

Notes:

- `--new-session-with-layout` is required so the script creates separate sessions even if run from inside Zellij.
- The session existence check strips the ` [Created ...]` suffix so session names containing spaces still compare correctly.
- The inside-Zellij branch uses `zellij action switch-session` because attaching from inside an existing Zellij session is not the same as attaching from a plain terminal.
